## Editor Props Interface

All FMD editors conform to this contract:

```ts
interface SectionEditorProps {
  section: FmdSection;              // The full FMD section entity
  project: Project;                 // Parent project data
  onSave: (content: Record<string, unknown>) => Promise<void>; // Save callback
  saving: boolean;                  // Loading state for save operation
}
```

**Example Registration Pattern**:

```ts
registerEditor(sectionType, EditorComponent);
```

## Implementation Pattern Summary

```typescript
// Typical editor structure (all 17 follow this pattern)

interface SectionEditorProps {
  section: FmdSection;      // Current section entity from DB
  project: Project;         // Parent project data
  onSave: (...);            // Save to API with optimistic concurrency
  saving: boolean;          // For save button state
  
  onReset?: () => void;     // Optional: refresh/clear data handler (from M10.6)
}

// Editor renders:
// 1. Schema wrapper parseFmdSectionContent(section.content)
// 2. Zod schema validation for section-type specific data
// 3. Form/table inputs per schema fields
// 4. Save button that calls onSave() with updated content
```
